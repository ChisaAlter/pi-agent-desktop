# Pi Desktop UI 重新设计文档

## 设计方向

基于 OpenAI Codex App 真实界面截图，采用**浅灰白色主题**，非深色主题。

## 布局结构（四栏）

```
┌────┬────────┬──────────────────────────┬──────────┐
│    │        │                          │ 悬浮进度  │
│图标 │ 项目   │     聊天区域              │   面板   │
│栏   │ 面板   │                          │ (可折叠) │
│48px│ 220px  │        自适应              │  280px   │
│    │        │                          │          │
└────┴────────┴──────────────────────────┴──────────┘
```

### 1. 左侧图标栏 (48px)
- 纯图标按钮，无文字标签
- 功能：新对话、搜索、插件、自动化、设置
- 白色背景，右侧细边框 `#e5e5e5`
- 悬停：`#f0f0f0` 背景

### 2. 左侧面板 (220px)
- **项目列表**：文件夹图标 + 项目名 + 时间/展开箭头
- **对话列表**：消息图标 + 对话名
- 白色背景，右侧细边框
- 当前项：`#f0f0f0` 背景

### 3. 主聊天区域（自适应）
- **顶部标题栏**：展开/折叠箭头 + 项目名 + 更多按钮 + 全屏按钮
- **消息列表**：AI消息（黑色圆形头像 + 文字 + 命令卡片）
- **命令结果卡片**：绿色勾选图标 + "已运行 X 条命令" + 代码块
- **底部输入区**：圆角输入框 + 附件按钮 + "完全访问权限"下拉 + 模型选择器 + 黑色圆形发送按钮

### 4. 右侧悬浮进度面板 (280px，可折叠)
- **悬浮样式**：固定定位，圆角，阴影，毛玻璃效果
- **切换按钮**：屏幕右侧中间圆形按钮
- **内容分区**：
  - 进度：勾选/时钟图标 + 任务描述
  - 输出：链接图标 + URL列表
  - 来源：文字说明

## 组件设计细节

### AI消息
```
[黑色圆形头像] AI
              文字内容...
              
              [命令结果卡片]
```

### 命令结果卡片
- 顶部：绿色勾选图标 + "已运行 X 条命令"
- 内容：等宽字体代码块
- 边框：`#e5e5e5`，圆角

### 输入区域
- 圆角边框输入框
- 左下角：+ 按钮、"完全访问权限"下拉
- 右下角："custom" 文本、"5.5 高" 文本、黑色圆形发送按钮

### 悬浮面板
- `position: fixed`
- `right: 16px; top: 64px; bottom: 16px`
- `width: 280px`
- 圆角、大阴影、毛玻璃 (`backdrop-blur`)
- 可折叠（滑入/滑出动画）

## 动画效果

1. **悬浮面板切换**：0.3s cubic-bezier 滑入/滑出
2. **切换按钮脉冲**：2s 无限脉冲动画
3. **悬停效果**：所有按钮 0.15s 背景色过渡
4. **面板展开/收起**：transform + opacity 动画

## 实施优先级

1. **第一步**：修改全局样式为浅色主题（`globals.css`、`tailwind.config.js`）
2. **第二步**：重构 `App.tsx` 为四栏布局
3. **第三步**：实现左侧图标栏 + 项目面板
4. **第四步**：重构聊天区域（消息、命令卡片、输入区）
5. **第五步**：实现右侧悬浮进度面板
6. **第六步**：添加动画和交互细节

## 注意事项

- **不要使用深色主题**：截图明确是浅灰白色
- **不要毛玻璃过度**：只在悬浮面板使用轻微 backdrop-blur
- **边框要细**：使用 `#e5e5e5` 1px 边框分隔区域
- **圆角要小**：面板和卡片使用 `rounded-lg` (8px) 或 `rounded-md` (6px)
- **阴影要克制**：只在悬浮面板使用明显阴影
