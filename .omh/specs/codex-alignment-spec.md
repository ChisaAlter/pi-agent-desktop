# Pi Desktop → Codex App 对齐规格书

## 状态: confirmed
## 日期: 2026-05-31

## 1. 背景

Pi Desktop 是一个基于 Electron 的 Windows 桌面应用，为 Pi Agent CLI 提供图形界面。
目标是对齐 OpenAI Codex App 的用户体验，使其成为一个专业的多代理编码工作台。

## 2. Codex App 核心特征（参照物）

### 2.1 布局结构
```
┌─────────────────────────────────────────────────────────┐
│ 顶部栏: 项目名 · 工作树选择器 · 终端按钮 · 设置          │
├────────┬──────────────────────────────┬─────────────────┤
│        │                              │                 │
│ 项目   │    聊天/线程区域              │   任务侧边栏     │
│ 侧边栏 │                              │   - 计划         │
│        │  消息 + 代码diff + 工具调用    │   - 来源         │
│ 线程   │                              │   - 生成文件      │
│ 列表   │                              │   - 进度         │
│        │                              │                 │
│ 220px  │         自适应                │   280px         │
├────────┴──────────────────────────────┴─────────────────┤
│ 内置终端 (可折叠, 高度可调)                               │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心功能清单
1. **多线程并行代理** — 每个线程是独立的代理实例，可并行运行
2. **Git Worktree 支持** — 隔离的工作环境，避免文件冲突
3. **内置终端** — 项目/工作树范围的终端
4. **代码 Diff 视图** — 文件变更的内联 diff 展示
5. **任务侧边栏** — 实时显示代理计划、来源、生成物
6. **代码审查面板** — 查看变更、审批/拒绝
7. **PR 审查集成** — GitHub PR 评论
8. **自动化模板** — 周期性任务模板
9. **Skills 系统** — 自定义代理行为
10. **云端环境** — 云端容器运行代理

## 3. 差距分析（Pi Desktop 现状 vs Codex App）

### 3.1 已实现（基础框架）
- [x] Electron + React + TypeScript 基础架构
- [x] 浅色主题（接近 Codex 的灰白配色）
- [x] 四栏布局框架（图标栏 + 项目面板 + 聊天 + 悬浮面板）
- [x] 基础聊天界面（消息气泡、Markdown 渲染）
- [x] 工具调用卡片（展开/折叠、状态指示）
- [x] 工作区管理（创建、切换、删除）
- [x] 会话管理（创建、切换、删除）
- [x] Pi CLI 集成（--print 管道模式）
- [x] Git 状态显示（分支、变更文件）
- [x] 基础设置面板（模型选择、Provider 配置）
- [x] Zustand 状态管理
- [x] IPC 通信架构

### 3.2 需要重大改进
- [ ] **线程管理** — 当前只有简单会话，缺乏线程概念（独立代理实例、状态追踪）
- [ ] **代码 Diff 视图** — 无 diff 渲染，工具调用只显示 JSON
- [ ] **任务侧边栏** — 当前悬浮面板是静态的，需要实时任务追踪
- [ ] **内置终端** — 完全缺失
- [ ] **代码审查流程** — 无变更审批机制
- [ ] **Worktree 支持** — 无 Git worktree 管理

### 3.3 需要优化
- [ ] **聊天体验** — 消息流式渲染需优化，工具调用可视化需增强
- [ ] **侧边栏** — Sidebar 组件使用旧的暗色主题，需统一为浅色
- [ ] **图标栏** — 功能不够明确，需要更好的视觉反馈
- [ ] **响应式布局** — 面板折叠/展开需要更流畅
- [ ] **快捷键** — 缺乏键盘快捷键支持

## 4. 实施计划（按优先级排序）

### Phase 1: UI 主题统一与基础体验优化
**目标**: 统一全应用的浅色主题，修复不一致的组件

1. **统一 Sidebar 组件为浅色主题**
   - 当前 Sidebar.tsx 使用暗色系（bg-gray-800, text-gray-300）
   - 对齐 DESIGN.md 中的浅灰白色配色
   - 更新所有硬编码的颜色值

2. **优化 IconBar 视觉反馈**
   - 当前图标栏功能按钮不够明确
   - 添加活跃状态指示器
   - 优化悬停效果

3. **改进 ChatInput 输入体验**
   - 支持多行输入自动扩展
   - 添加快捷键提示（Enter 发送，Shift+Enter 换行）
   - 优化附件按钮和权限选择器的交互

4. **优化 MessageBubble 样式**
   - 用户消息和 AI 消息的视觉区分
   - 改进 Markdown 渲染质量
   - 添加代码块的复制按钮

### Phase 2: 代码 Diff 视图与工具调用增强
**目标**: 实现类似 Codex 的文件变更可视化

1. **实现 DiffViewer 组件**
   - 支持 unified diff 和 split diff 视图
   - 语法高亮
   - 行号显示
   - 添加/删除/修改行的颜色标记

2. **增强 ToolCallCard**
   - 对 read/write/edit 工具调用，自动渲染 diff
   - bash 命令显示终端风格的输出
   - 添加文件路径面包屑导航

3. **实现变更文件列表**
   - 类似 Codex 的文件树视图
   - 每个文件显示变更统计（+N -N）
   - 点击文件跳转到对应的 diff

### Phase 3: 任务侧边栏与进度追踪
**目标**: 实现实时任务追踪面板

1. **重设计 FloatingPanel 为 TaskSidebar**
   - 显示代理当前执行计划
   - 实时更新进度步骤
   - 显示引用的文件/来源
   - 显示生成的文件列表

2. **实现进度状态机**
   - pending → running → completed/failed
   - 每个步骤可展开查看详情
   - 错误步骤显示错误信息

3. **添加任务操作**
   - 暂停/恢复任务
   - 取消任务
   - 查看任务日志

### Phase 4: 内置终端集成
**目标**: 添加项目范围的内置终端

1. **集成 xterm.js**
   - 终端模拟器
   - 支持基本的 shell 操作
   - 可折叠、高度可调

2. **终端与代理联动**
   - 代理执行的命令在终端中可见
   - 可以手动在终端中执行命令
   - 终端输出可作为上下文发送给代理

### Phase 5: 线程管理与多代理并行
**目标**: 实现类似 Codex 的线程系统

1. **线程模型**
   - 每个线程 = 独立的代理实例
   - 线程有状态：idle, running, completed, failed
   - 线程有独立的消息历史

2. **线程列表 UI**
   - 在侧边栏显示线程列表
   - 线程状态指示器
   - 创建/删除/切换线程

3. **并行执行**
   - 多个线程可同时运行
   - 线程间隔离（文件系统、上下文）

### Phase 6: Git Worktree 支持
**目标**: 隔离的工作环境

1. **Worktree 管理 UI**
   - 创建/删除 worktree
   - 切换 worktree
   - 显示 worktree 状态

2. **Worktree 与线程绑定**
   - 每个线程可绑定到一个 worktree
   - 自动创建 worktree 用于新线程

## 5. 技术约束

- Electron 34 + React 19 + TypeScript
- 使用 pnpm monorepo
- 所有颜色必须遵循 DESIGN.md 配色方案
- 组件必须使用 Tailwind CSS
- 状态管理使用 Zustand
- 保持现有的 IPC 架构

## 6. 验收标准

- [ ] 所有组件使用统一的浅色主题
- [ ] 代码变更以 diff 形式可视化
- [ ] 任务进度实时显示在侧边栏
- [ ] 内置终端可正常使用
- [ ] 可以创建和管理多个线程
- [ ] 支持 Git worktree 隔离
- [ ] 所有功能有对应的键盘快捷键
- [ ] 应用整体视觉效果接近 Codex App
